## 2024-06-27 - [Optimize .find() inside loop]
**Learning:** In Cloudflare Workers running V8, using `Array.prototype.find()` with complex closures and repeated method calls (like `.toLowerCase()`) on thousands of items inside a tight loop is surprisingly slow compared to a raw `for...of` loop with pre-computed values. Creating Maps or Sets can sometimes be slower due to the overhead of allocation and iterating to build them, unless you're doing many lookups. For a single pass to find an item, an optimized `for...of` loop with pre-computed comparison strings is significantly faster (3-4x speedup).
**Action:** When searching an array inside an endpoint, use a simple `for...of` loop and extract `.toLowerCase()` or other static operations outside the loop for the target value. Avoid `.find()` closures if performance is critical or arrays are moderately large.
