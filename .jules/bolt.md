## 2024-06-18 - Optimize Genre Aggregation Loops
**Learning:** Using a single pass and a local `Set` or `Map` to track duplicates *during* the inner loop is more efficient in V8/Node than instantiating a reusable Set, calling `.clear()`, filling it, and then performing a second iteration loop over it. Furthermore, directly updating properties on a referenced object from a `Map.get()` is faster and cleaner than repeatedly calling `Map.set()` in a loop.
**Action:** When tracking unique occurrences in inner loops for aggregation, check existence and update the accumulator synchronously within the same loop pass.
