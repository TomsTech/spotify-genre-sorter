## 2025-01-08 - Accessible Responsive UI States
**Learning:** When responsive UI elements dynamically change their visual state based on viewport size (e.g., auto-collapsing a sidebar on mobile viewports), their corresponding ARIA attributes (such as `aria-expanded`) must be explicitly initialized in JavaScript during component load to match that responsive state. Relying solely on toggle interaction events to update these attributes causes the DOM and accessibility tree to be out of sync upon initial load. Additionally, ARIA boolean attributes must be set using explicit string values (`'true'` or `'false'`) rather than JavaScript booleans.
**Action:** Always ensure initial load scripts that handle responsive states also sync the related ARIA attributes using string values.
