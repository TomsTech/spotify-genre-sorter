## 2026-05-18 - Dynamic aria-expanded value **Learning:** When dynamically setting the `aria-expanded` attribute with `setAttribute` in JavaScript, passing a boolean (e.g. `!isCollapsed`) results in invalid HTML (e.g., `aria-expanded=""` or omitted entirely depending on the browser), which causes the attribute to fail screen reader validation. **Action:** Always ensure the value is explicitly converted to and passed as a string (e.g., `isCollapsed ? 'false' : 'true'`).
