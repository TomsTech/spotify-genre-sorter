## 2026-05-16 - Explicit String Values for ARIA Attributes **Learning:** When dynamically setting ARIA attributes like `aria-expanded` via JavaScript, passing boolean values (e.g., `!isCollapsed`) can lead to invalid HTML semantics because they may evaluate implicitly rather than explicitly serializing to `'true'` or `'false'`. Additionally, collapsible panels require `aria-controls` explicitly defining the linked panel. **Action:** Always pass explicit strings (`'true'` or `'false'`) when calling `setAttribute('aria-expanded', ...)` and always provide an `aria-controls` pointing to the controlled element's ID for toggle buttons.
